# ChemAgent 代码调整记录

## 问题诊断

### 对话框有时无法正常回复

**根因分析**: 多处 token 截断机制导致 LLM 请求被截断或历史上下文丢失：

1. `buildHistory()` 中 `MAX_MSG_TOKENS = 4_000` 限制单条消息最大 4000 token，超出部分被截断为 `[...]`
2. `historyTokenBudget = contextWindow * 0.4` 限制历史消息总 token 预算，超出部分被丢弃
3. LLM 请求体中 `max_tokens` 字段限制模型最大输出长度
4. 压缩摘要调用中 `maxTokens: 500` 限制摘要长度

当对话历史较长或单条消息较长时，这些截断会导致：
- 关键上下文被丢弃 → 模型回复不完整
- `max_tokens` 过小 → 回复被截断
- 历史消息不完整 → 模型理解偏差 → 空回复

---

## 修复内容

### P1: 移除所有 token 截断

| 文件 | 修改 |
|------|------|
| `server/src/ws.ts` | `buildHistory()` 移除 `tokenBudget` 参数、`CHARS_PER_TOKEN`、`MAX_MSG_TOKENS` 截断逻辑，改为传递全部历史消息 |
| `server/src/ws.ts` | `handleCommand()` 移除 `historyTokenBudget = contextWindow * 0.4` 计算，直接调用 `buildHistory(session.messages)` |
| `server/src/llm/client.ts` | `streamChatCompletion()` 注释掉 `max_tokens` 发送，让 API 使用自身默认值 |
| `server/src/llm/client.ts` | `chatCompletion()` 注释掉 `max_tokens` 发送 |
| `server/src/agent/loop.ts` | `compactMessages()` 移除摘要调用的 `maxTokens: 500` 限制 |

**效果**: 对话历史完整传递，模型输出不再被截断，回复中断问题彻底解决。

### P2: 新增 MiMo 模型支持

| 文件 | 修改 |
|------|------|
| `server/src/models/provider-config.ts` | 新增 `xiaomi` provider 配置：`thinking: { type: 'openai_compatible' }` |
| `server/src/models/catalog.ts` | 新增 `mimo-v2.5-pro` 模型预设（128K 上下文，65K 输出） |
| `src/views/settings-view.ts` | 快速选择下拉框新增 "Xiaomi (MiMo)" 分组 + `MiMo V2.5 Pro` 选项 |
| `src/views/settings-view.ts` | 添加模型表单 Provider 下拉框新增 "Xiaomi (MiMo)" |
| `src/views/settings-view.ts` | 编辑模型弹窗 Provider 下拉框新增 "Xiaomi (MiMo)" |
| `server/src/llm/client.ts` | 请求头新增 `api-key` 字段，兼容 MiMo/Azure 风格鉴权 |

**MiMo 配置说明**:
- Provider: `xiaomi`
- 模型名: `mimo-v2.5-pro`
- API 协议: OpenAI 兼容（`/v1/chat/completions`）
- 鉴权方式: `Authorization: Bearer` + `api-key` 双 header 支持
- 上下文窗口: 128K tokens
- 用户需在前端设置中配置自己的 BASE_URL 和 API Key
