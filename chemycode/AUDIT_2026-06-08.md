## 一、Token 截断审计 — 结论：✅ 已全部清除

### 1.1 已移除的截断

| 位置 | 原问题 | 修复 |
|------|--------|------|
| `ws.ts` `buildHistory()` | `MAX_MSG_TOKENS=4000` 截断单条消息 | 移除，传递完整消息 |
| `ws.ts` `handleCommand()` | `historyTokenBudget = contextWindow * 0.4` 限制历史总量 | 移除，传递全部历史 |
| `llm/client.ts` `streamChatCompletion()` | `max_tokens` 发送到 API | 注释掉，API 使用自身默认值 |
| `llm/client.ts` `chatCompletion()` | `max_tokens` 发送到 API | 注释掉 |
| `agent/loop.ts` `compactMessages()` | 摘要调用 `maxTokens: 500` | 移除 |
| `agent/loop.ts` 摘要构建 | `.slice(0, 200)` 截断工具结果和消息 | 改为 `.slice(0, 2000)` |

### 1.2 保留的安全截断（合理，不影响对话）

| 位置 | 限制 | 原因 |
|------|------|------|
| `tools/bash-exec.ts` stdout/stderr | 100KB | 防止 `ls -R /` 等命令输出撑爆 LLM context |
| `tools/bash-exec.ts` details | 1000 chars | 仅元数据展示，不影响 LLM 输入 |
| `tools/file-read.ts` | 1MB | 防止读取超大文件 |
| `tools/file-write.ts` | 10MB | 防止写入超大文件 |
| `llm/client.ts` 错误信息 | 200 chars | 仅错误展示 |
| `routes/models.ts` 测试错误 | 100 chars | 仅错误展示 |

### 1.3 现行 maxTokens 字段状态

- `catalog.ts` 中的 `maxTokens` 值 → **仅作为 UI 默认值展示**，不会发送到 API
- `ws.ts:132` 的 `model.maxTokens` → 存入 `LLMClientConfig`，但 **不会出现在请求体中**（已被注释）
- `routes/models.ts:60` 的 `maxTokens ?? defaults.maxTokens` → 存入 store，**仅用于前端显示**

**结论：API 请求体中不再包含 `max_tokens` 字段，模型输出不会被截断。**

## 二、Skill 接口审计 — 结论：✅ 功能完整

### 2.1 Skill 生命周期

| 功能 | 端点 | 状态 |
|------|------|------|
| 列表 | `GET /api/skills` | ✅ 正常，含 toolCount/toolNames |
| 安装 | `POST /api/skills/:id/install` | ✅ 热加载工具到 registry |
| 卸载 | `POST /api/skills/:id/uninstall` | ✅ 从 registry 注销工具 |
| 删除 | `DELETE /api/skills/:id` | ✅ 删除文件 + 注销工具 + 移除 store |
| 导入 | `POST /api/skills/import` | ✅ zip 解压 + manifest 解析 + 热加载 |
| 多 skill 导入 | 同上 | ✅ 自动检测多个 manifest + 按名称去重 |

### 2.2 Skill 工具注册流程

```
zip 上传 → extractZip() → findAllManifests() → importSingleSkill()
  → loadSingleSkill() → registerSkillTool() → toolRegistry.register()
  → agent loop 通过 toolRegistry.getLLMToolsFiltered() 获取所有可用工具
  → LLM 可调用 skill 注册的工具
```

### 2.3 Skill 工具执行流程

```
LLM 返回 tool_calls → agent loop 解析工具名和参数
  → toolRegistry.execute() → 权限检查 → 参数校验 → 执行 handler
  → 结果返回 LLM → LLM 继续生成
```
