## 调整清单002 — 7 项修复

### P1: 移除最大工具调用轮次限制

**原问题**: `MAX_TOOL_ROUNDS = 10` 导致复杂任务在第 10 轮后被强制终止。

**修复**: 移除 `MAX_TOOL_ROUNDS` 常量，`while` 循环改为无限循环，仅在 LLM 返回 `finish_reason=stop` 或 abort signal 触发时退出。

| 文件 | 修改 |
|------|------|
| `server/src/agent/loop.ts` | 移除 `MAX_TOOL_ROUNDS`，`while (round < maxToolRounds)` → `while (true)` |
| `server/src/agent/types.ts` | 移除 `maxToolRounds` 字段 |

### P2: 工具执行内容可见

**原问题**: `tool_call_end` 的 `result` 被截断为 500 字符，前端看不到工具输出。

**修复**:

| 文件 | 修改 |
|------|------|
| `server/src/agent/loop.ts` | `result: toolResult.content.slice(0, 500)` → `result: toolResult.content`（完整输出） |
| `src/state.ts` | `handleStreamEvent` 的 `tool_call_end` 分支：将 `ev.result` 存入 `m.content` |
| `src/views/chat-view.ts` | tool 消息传递 `m.content` 作为 result |
| `src/components/tool-call-card.ts` | 工具完成/失败时自动展开详情面板 |

### P3: 前端停止按键

**原问题**: 无法中止正在执行的任务。

**修复**:

| 文件 | 修改 |
|------|------|
| `src/state.ts` | 新增 `stopGeneration()` — 发送 `cancel` 命令到后端 |
| `src/views/chat-view.ts` | 流式生成时显示红色停止按钮 ■，点击调用 `stopGeneration()` |

**停止流程**: 用户点击 ■ → `stream.send({type:'cancel', sessionId})` → 后端 `abortCtrl.abort()` → LLM 请求中断 → 工具执行中断 → 发送 `error` 事件

### P4: bash_exec 成功率提升

**原问题**: Windows 上使用 `cmd` 执行命令，复杂命令（含特殊字符、管道等）经常失败。

**修复**:

| 文件 | 修改 |
|------|------|
| `server/src/tools/bash-exec.ts` | 默认 shell 从 `cmd` 改为 `powershell`，参数从 `['/c', command]` 改为 `['-NoProfile', '-NonInteractive', '-ExecutionPolicy', 'Bypass', '-Command', command]` |
| 同上 | 默认超时从 30s 提升到 60s |

### P5: 对话记录留存

**原问题**: 点击 Skills 后再新建任务，之前的对话丢失。

**修复**: 对话列表持久化在侧栏，切换视图不影响已有对话。

| 文件 | 修改 |
|------|------|
| `src/state.ts` | 新增 `sessions` 状态字段 + `createNewSession()` / `switchSession()` / `refreshSessions()` / `deleteSession()` |
| `src/components/sidebar.ts` | 新增「对话列表」区域，显示所有会话，点击切换 |

### P6: 模型选项初始化

**原问题**: 模型选择器显示"未配置模型"，需要发消息后才刷新。

**修复**:

| 文件 | 修改 |
|------|------|
| `src/views/chat-view.ts` | `connectedCallback` 中检测模型为空时调用 `ensureModelsLoaded()` 直接请求 API |

### P7: 多对话框并行

**原问题**: 只有"最近任务"，没有独立对话框列表。

**修复**: 侧栏新增「对话列表」区域，支持新建、切换、删除对话。每个对话有独立的 session 和消息上下文。

| 文件 | 修改 |
|------|------|
| `src/components/sidebar.ts` | 新增对话列表 UI + 新建对话按钮 + 删除按钮 |
| `src/state.ts` | `loadAll()` 加载 sessions 列表；`createNewSession()` 创建并切换；`switchSession()` 加载历史 |

## 消息元数据显示（新增）

**需求**: 每条 agent 消息下方显示 token 使用信息，格式类似 OpenClaw：
```
↑1.4k  ↓350  R164.4k  16% ctx  deepseek-v4-pro
```

### 实现

| 文件 | 修改 |
|------|------|
| `server/src/llm/types.ts` | `LLMStreamCallbacks` 新增 `onUsage`；`LLMCompletionRequest` 新增 `stream_options` |
| `server/src/llm/client.ts` | 请求体添加 `stream_options: { include_usage: true }`；SSE 解析中捕获 `chunk.usage` |
| `server/src/agent/loop.ts` | 新增 `AggregatedUsage` 类型；多轮累加 usage；返回 `{ content, usage }` |
| `server/src/agent/types.ts` | 无变化（返回类型在 loop.ts 中） |
| `server/src/ws.ts` | done 事件包含 `usage` 和 `model` 字段 |
| `server/src/types.ts` | `ChatMessage` 新增 `usage` 和 `model`；done 事件新增 `usage` 和 `model` |
| `src/types.ts` | `ChatMessage` 新增 `usage` 和 `model` |
| `src/api/types.ts` | `StreamEventDone` 新增 `reasoningTokens` 和 `model` |
| `src/state.ts` | `handleStreamEvent` 的 done 分支存储 usage/model 到消息 |
| `src/views/chat-view.ts` | 新增 `renderUsageMeta()` 方法 + `.msg-meta` / `.meta-item` / `.meta-model` 样式 |

### 数据流

```
LLM API (stream_options.include_usage=true)
  → SSE last chunk: { usage: { prompt_tokens, completion_tokens, total_tokens } }
  → llm/client.ts: onUsage({ promptTokens, completionTokens, totalTokens })
  → agent/loop.ts: aggregatedUsage += usage (多轮累加)
  → ws.ts: done event { usage: { promptTokens, ... }, model: "deepseek-v4-pro" }
  → state.ts: updateMessage(agentMsgId, { usage, model })
  → chat-view.ts: renderUsageMeta(usage, model)
    → ↑1.4k ↓350 R164.4k 16% ctx deepseek-v4-pro
```

| 项目 | 说明 |
|------|------|
| 存储位置 | `data/apikeys.json`（独立于用户数据） |
| 写入 | POST/PATCH models → apiKeyStorage.set() |
